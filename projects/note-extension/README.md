# note 自動投稿ボット（Chrome拡張機能）

あなたのChromeの中で動いて、**Geminiで記事を生成 → note に自動投稿**します。
あなたのChromeは note にログイン済みなので、**Cookieもログイン情報の受け渡しも不要**。
「Startを押すと、一定間隔で勝手に投稿してくれる」拡張機能です。

## 入れ方（5分・ターミナル不要）

1. このフォルダ `projects/note-extension/` をPCに用意（リポジトリをDL or clone）
2. Chromeで `chrome://extensions` を開く
3. 右上の「**デベロッパーモード**」をON
4. 「**パッケージ化されていない拡張機能を読み込む**」→ `note-extension` フォルダを選ぶ
5. ツールバーにアイコンが出る → クリックして設定パネルを開く

## 使い方

1. **Gemini APIキー**を入力（https://aistudio.google.com/apikey）
2. アカウント名（既定 `light_roses761`）、テーマは `auto` のままでOK
3. **まず「投稿モード=下書き」**にして「**今すぐ1回**」を押す
   → note に下書きが作られるか確認（タブが開いて自動で入力されます）
4. うまくいったら「投稿モード=公開」に変更
5. 「**▶ 開始**」を押すと、設定した間隔で自動投稿。**Chromeを開いている間ずっと**動きます（1日上限あり）
6. 止めるときは「**■ 停止**」

## 設定
| 項目 | 意味 |
|------|------|
| Gemini APIキー | 記事生成に使用 |
| テーマ | `auto`=アカウントから自動判定 / 文字列で固定 |
| 1記事の文字数 | 既定20000 |
| 1日の上限 | **凍結対策。低めから（例3）** |
| 実行間隔(分) | 自動実行の間隔 |
| 投稿モード | 下書き / 公開 |

## 正直な注意
- noteには公式投稿APIが無いため、投稿は**編集画面のDOM操作**です。noteのUIが変わると
  `background.js` の `postInPage()` 内のセレクタ調整が必要なことがあります
  （手元のClaude Codeに「note編集画面に合わせてセレクタ直して」と頼むと早いです）。
- **AI記事の大量・即公開はアカウント凍結リスク**。最初は下書き＋上限低めで様子見を。
- サムネ画像の自動生成・添付はこの版では未対応（テキスト記事のみ）。必要なら追加できます。
- Chromeを閉じている間／スリープ中は動きません（「Chrome起動中ずっと」が条件）。
